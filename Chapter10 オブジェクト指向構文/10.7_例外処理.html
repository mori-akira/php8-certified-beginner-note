10.7_例外処理

例外クラスの種類

  • Exception: 全ての例外の基底クラス
  • ErrorException: エラー例外
  • LogicException: ロジック誤り時の例外
  • RuntimeException: 実行時の例外

マルチキャッチ


<?php
try {
    ...
} catch (HogeException | FugaException $ex) {
    // HogeException、FugaExceptionをハンドリング
    ...
}

例外を送出


<?php
// RuntimeExceptionを送出
throw new RuntimeException();

assert命令


<?php
function hoge(mixed $param) {
    assert($param != null);
    return $param . $param;
}
print(hoge('hgoe'));
print(hoge(null));

カスタム例外を定義する場合の作法

  1. 標準のExceptionクラス、またはそのサブクラスを継承する
  2. クラス名の接尾辞はException
  3. アプリが複数の例外を持つ場合は、単一のカスタム例外を意味するインターフェイスを定義し、これを実装する
