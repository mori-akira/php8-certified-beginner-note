2.3_データ型

データ型の種類

分類 データ型 説明
スカラー型 論理型(bool) true/false
整数型(int) 10進数、16進数(0xから始まる)、8進数(0から始まる)
浮動小数点型(float、double) 1.8×10308が最大値
文字列型(string) 文字列
複合型 配列型(array) 配列
オブジェクト型(object) データと手続きの集合
コールバック型(callback) 一連の手続きの集合
特殊型 リソース型(resource) 外部リソースへの参照
NULL型 null

論理リテラル

true/falseは、大文字/小文字を区別しない

falseと判定される値

  1. 空文字
  2. 整数の0
  3. 浮動小数点数の0
  4. 空配列
  5. null

浮動小数点数の指数表現リテラル

次のように表現する。
(仮数部)e(符号)(指数部)
これは次を意味する。
(仮数部)×10(符号部)(指数部) 


<?= 0.01 == 1e-2 ? 'OK' : 'NG' ?>

<?= 80000 == 8e+5 ? 'OK' : 'NG' ?>

出力結果


OK

NG

数値セパレータ

php7.4以降は、数値リテラルの可読性向上のため数値のセパレータとして_(アンダースコア)を記述できる。

文字列リテラル

  • シングルクォートまたはダブルクォートで囲む
  • エスケープするときはバックスペース

シングルクォートとダブルクォートによる文字列の違い

  1. ダブルクォート　: {$変数名}のフォーマットで、プレースホルダを使用できる
  2. シングルクォート: 「\\」、「\'」のエスケープシーケンスを使用できる
    ダブルクォート　: すべてのエスケープシーケンスを使用できる

使用可能な主なエスケープシーケンス

エスケープシーケンス 説明
\r キャリッジリターン
\n ラインフィード
\t 水平タブ
\v 垂直タブ
\f フォームフィード
\$ ダラー記号
\\ バックスラッシュ
\" ダブルクォート
\nnn 3桁の8進数nnnで表される文字
\xnn 2桁の16進数nnで表される文字
\u{nnnn} 4桁の16進数nnnnで表されるUnicode文字

ヒアドキュメント

ヒアドキュメントの開始文字列

  • <<<EOD変数のプレースホルダやエスケープシーケンスを展開する
  • <<<"EOD"変数のプレースホルダやエスケープシーケンスを展開する
  • <<<'EOD'変数のプレースホルダやエスケープシーケンスを展開しない

ヒアドキュメントの終了文字列

EOD;

※EODは変数名として指定できる任意の文字列に置き換えることができるが、一般的に大文字を使用する。

ヒアドキュメントのインデントの除去

終了文字列よりインデントが浅い行があると、Parse errorが発生する。
